= Rebrickable Refined
:toc-title:
:toc:

=== Browser extension that enhances https://rebrickable.com/[rebrickable.com] with new features and fixes some annoyances

"Rebrickable Refined" is a browser extension is meant to improve your experience of using https://rebrickable.com/[rebrickable.com]. It is *not affiliated* with https://rebrickable.com/[rebrickable.com]

The features are all something that I personally wish was part of the main website. I spent a lot of time developing them for myself, so I can only hope these improvements will reach as many people as possible.

You can see all the features described below. +
Per your preference, they can all be individually enabled or disabled in the Settings screen (click on the extension's icon in the toolbar). But in my personal opinion, all of them are good to enable -- it should be rare for any of the features to disrupt your experience.

= Privacy

This extension never sends your data elsewhere *other than the following exceptions* where it uses *external services*:

* For the features "Add an improved parts search function..." and "Replace the parts search function...":
+
It sends the search queries that you type into the search field to the external https://setmaster.pryp.in/["SetMaster"] service, so that the service can return the search results for that query. Only the exact search query that you typed is sent. This external service is owned by me. It stores access logs (which include all incoming queries) for up to 3 months. (See https://setmaster.pryp.in/privacy/[full privacy policy])

** As such, the "Replace the parts search function" setting is disabled by default, you should enable it in Settings.
** Note that the "Add an improved parts search function" setting is *enabled* by default, but it only adds a new menu item, so no data can be sent until you choose the "Parts+" menu item.

* For the feature "Show additional related parts...":
+
It periodically downloads part definition updates from a static page hosted on GitHub. (No data is sent.)

All external-facing features (present and future) will be described in this section. They will also always require some kind of opt-in to use.

== Main search field

[#add-main-parts-search]
=== Add an improved parts search function to the main search field ("Parts+")

[frame=none,cols="a,a"]
|===
|Before: +
image:docs/add-main-parts-search-before.png[]
|After: +
image:docs/add-main-parts-search-after.png[]
|===

This is completely new search functionality based on an external service that I've built.

* The search favors precise matches of whole words and phrases.
* It tends to show most common parts first. Due to this, printed parts don't clutter the search.
* Some fuzzy search ability is still present, in particular it knows about plurals and many synonyms. Some slang keywords like "SNOT" are supported.
* It has specific understanding of dimensions like "2x1" so that it also finds parts that have "1x2" or "1x2x5".
* It directly downranks "Duplo", "Quatro", "Primo" and "Modulex" parts unless that word is explicitly mentioned. Similarly for stickers and cards.

This functionality concerns only searching of *parts*. +
It is activated only when you select the new "Parts+" dropdown item instead of the old "Parts" item.


[#focus-main-parts-search]
=== Always pre-select the new search function when viewing any parts-related page

When visiting parts search or a part's details page, the top-right global Search field will always have "Parts+" selected by default.

[#remember-selected-search-option]
=== Remember the selected search category across page visits

When you make a new selection in the top-right global Search field, the selection is remembered across pages.

=== Filter selectable part colors to only the existing ones in the "Add Part" dialog

When adding a new part, you first find the correct part and then select its color. The color selection normally shows all colors. This changes that - now the colors are limited to only the existing colors of the part that you just found in the search results.

[#part-dialog-always-sort-colors]
=== Always sort the color dropdown alphabetically in the "Use Part" dialog

When clicking on a part in order add it to another list, the color dropdown is sorted by how often that color occurs. For better "muscle memory", now we always sort the colors alphabetically instead.
